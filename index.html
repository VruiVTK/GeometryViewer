Vruivtk

Application demonstrating the use of vtkRenderingExternal module to render VTK objects in the VRUI context

Welcome to VruiVTK

This application demonstrates use of the new vtkRenderingExternal module for VRUI and VTK integration.

Supported Operating Systems

  • Linux
  • Mac OSX

Prerequisites

NOTE: To build VruiVTK along with all its prerequisites, simply build the superbuild from VruiVTKSuperbuild

VruiVTK application requires the following software to be built and installed.

  • CMake
  • Vrui
  • VTK with the vtkRenderingExternal module turned ON
  • TCLAP

Clone VruiVTK source

The VruiVTK application can be built from source obtained using one of the download links on this page or using the commandline as follows:

$ git clone https://github.com/VruiVTK/VruiVTK.git

The above command creates a directory VruiVTK with the source code.

Configure Instructions

Configure the build using CMake.

Set the following CMake variables:

* VRUI_PKGCONFIG - Path to pkgconfig directory of your VRUI installation
* VTK_DIR - Path to the VTK build directory
* TCLAP_INCLUDE_DIR - Path to include directory of your TCLAP installation

Once all the required variables are set, press generate to generate the makefile.

Build Instructions

Build using the make tool.

$ make

Run VruiVTK

The build step will create an executable VruiVTK under the build directory that can be used to run the application. To get usage instructions for the application:

$ VruiVTK --help

To render an OBJ file using VTK, use the '-f' or '--fileName' argument.

$ VruiVTK -f cow-nonormals.obj

The following demo video shows VTK rendering an OBJ dataset in the VRUI context. The dataset was obtained from here.

If no arguments are provided, the application will bring up a VRUI window with a vtkCube rendered at its origin.
